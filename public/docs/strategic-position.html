Drie dingen die bijna niemand doet

Prestatie 1

Een primitief gedefinieerd

Niet een feature. Niet een SaaS. Niet een tool. Maar een formeel gedefinieerde verificatie-eenheid:

V(B, P, L) → { valid | invalid | unverifiable }

Een primitive beschrijft een minimale, formeel begrensde eigenschap. Het is onafhankelijk van implementaties. Het overleeft de maker. Dat is precies wat IEC doet.

Prestatie 2

Het primitive institutioneel losgekoppeld

De meeste bedrijven bouwen een product, noemen het een standaard, en houden de verifier proprietary.

Hier is het anders:

  • De spec staat buiten het bedrijf — anchoring-spec.org (Public Domain)
  • De verifier is onafhankelijk — verify-anchoring.org (zero-CDN, zero-API)
  • De commerciële laag is gescheiden — umarise.com / Core API

Dat is infrastructuurdenken.

Prestatie 3

Infrastructuur gepubliceerd

Een infrastructuurlaag heeft drie kenmerken:

  • Overleeft de maker
  • Is publiek verifieerbaar
  • Heeft geen lock-in afhankelijkheid

verify-anchoring.org voldoet aan alle drie. De CLI-scripts (verify-anchor.sh, verify-anchor.py) werken volledig offline, zonder Umarise-infrastructuur.

Drie-track verificatie

VeldTypeBeschrijving

VeldTypeBeschrijving
origin_idUUIDFK → origin_attestations (1:1)

bitcoin_block_heightINTEGERBloknummer (wanneer verankerd)
anchored_atTIMESTAMPTZAnker-timestamp
Kritiek invariant: Geen kolom voor content bestaat in enige tabel. Dit is architectureel, niet beleid. Er bestaat geen mechanisme om bestandsinhoud op te slaan, zelfs als iemand dat zou proberen. De hash is een eenrichtingsfunctie.

Welke positie is dit

Niet: marktleider, standaard, of regulator-approved.

Wél: de eerste formeel gedefinieerde, publiek gescheiden anchoring primitive met onafhankelijke verifier, werkende Core API, en twee live implementaties die het bewijs leveren.

Wat er is en wat er niet is

Aanwezig

Fundament staat

  • Technisch fundament — Core API frozen, Bitcoin-ankering live
  • Semantisch fundament — IEC specificatie, formele verificatiefunctie
  • Governance-fundament — Publieke scheiding, Public Domain licentie
  • Twee live B2C-implementaties — anchoring.app + itexisted.app
  • Drie-track verificatie — In-app, referentie, CLI
  • Drie-laag bewijs — Temporeel + Device + Attestant

Nog niet aanwezig

  • Institutionele erkenning
  • Externe implementaties (partner pilots)
  • Juridisch precedent
  • Volume-adoptie

Eerlijk oordeel

Dit is geen hypepositie. Dit is geen marketingclaim. Dit is: een infrastructuur-weddenschap gebaseerd op een publiek primitive. Of het groot wordt, hangt af van adoptie. Maar het fundament klopt. En dat is zeldzaam.
Architectonisch sta je op het juiste niveau.

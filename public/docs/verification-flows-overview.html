Verification Flows

Vier tracks, drie doelgroepen, één waarheid.

Tracks

In-app Verify

itexisted.app/verify
Gebruiker — stateless convenience check
  • ZIP openen
  • certificate.json uitlezen
  • origin_id → registry lookup
  • SHA-256 hash vergelijken record-check
  • Bitcoin anchor status ophalen bestaans-check
  • Artifact hash-check Triple-Gate
  • VERIFIED · PENDING · ERROR
Afhankelijkheid: Umarise registry (online)

Core API Verify

umarise.com/verify
Partner / integrator — via Core API
  • SHA-256 hash invoeren of ZIP uploaden
  • v1-core-verify endpoint registry lookup
  • v1-core-resolve → origin metadata
  • v1-core-proof → OTS proof status bestaans-check
  • VERIFIED · PENDING · NOT FOUND
Publiek endpoint — geen API-key vereist
Afhankelijkheid: Umarise Core API (online)

ZIP Verifier

verify-anchoring.org
Iedereen — browser, zero-API
  • ZIP uploaden in browser
  • certificate.json uitlezen
  • Artifact localiseren
  • SHA-256 herberekenen integriteit
  • Vergelijken met certificate hash
  • .ots proof → lokale OTS library bestaan
  • Valid · Mismatch · Ledger confirmed
Afhankelijkheid: Geen

Hash + OTS Verifier

verify-anchoring.org
API-integrators — handmatige input
  • SHA-256 hash plakken
  • normalizeHash() sha256: / raw
  • .ots proof file uploaden
  • Lokale OTS library verificatie bestaan
  • Ledger-confirmed · Invalid
Afhankelijkheid: Geen

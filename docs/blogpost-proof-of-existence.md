# How to prove a file existed at a specific time

*dev.to / umarise.com/blog*

---

You have a file. A contract, a photo, a design, a dataset. You need to prove it existed before a specific moment, without relying on a third party someone could argue is biased.

This is called proof of existence. The underlying cryptography is simple. A clean developer primitive for it has been missing.

Here is one.

---

## Install

Node.js:

```bash
npm install @umarise/anchor
```

Python:

```bash
pip install umarise-core-sdk
```

---

## Anchor a file

Node.js:

```javascript
import { UmariseCore } from '@umarise/anchor';

const core = new UmariseCore({ apiKey: 'um_...' });
const origin = await core.attest('sha256:9f3a...');
console.log(origin.origin_id);
```

Python:

```python
from umarise import UmariseCore

core = UmariseCore(api_key="um_...")
origin = core.attest("sha256:9f3a...")
print(origin["origin_id"])
```

The file never leaves your machine. You compute a SHA-256 hash locally and anchor it into Bitcoin via OpenTimestamps. What comes back is an origin record with an `origin_id`.

---

## What the proof contains

The `.proof` file (generated by the CLI) is a ZIP containing:

```
certificate.json    ← origin_id, hash, timestamp
proof.ots           ← OpenTimestamps binary proof
```

The proof is verifiable by anyone, without an account, without trusting the issuer, using a block explorer or the open verifier at [verify-anchoring.org](https://verify-anchoring.org).

---

## CLI

The fastest path. No code needed:

```bash
npx @umarise/cli anchor contract-draft-v3.pdf
# ✓ hash computed: sha256:9f3a...
# ✓ anchored: origin_id abc4f2...
# ✓ proof saved: contract-draft-v3.pdf.proof
```

Verify (after Bitcoin confirmation, ~2 hours):

```bash
npx @umarise/cli verify contract-draft-v3.pdf.proof
# ✓ Hash Match | Bitcoin Block #884201 | 2026-03-05 | VALID
```

## CI/CD

Anchor every build artifact on publish:

```yaml
# .github/workflows/anchor.yml
- uses: AnchoringTrust/anchor-action@v1
  with:
    file: dist/release.tar.gz
  env:
    UMARISE_API_KEY: ${{ secrets.UMARISE_API_KEY }}
```

The `.proof` file is automatically uploaded as a build artifact.

---

## Why this is useful now

AI generates convincing content locally, without network traces, without cost. A document, a photo, a contract. The question has shifted from "is this fake?" to "can you prove when this existed?"

Anchoring gives you a fact that sits outside your own system. The hash is in Bitcoin. The block is public. The proof is yours.
